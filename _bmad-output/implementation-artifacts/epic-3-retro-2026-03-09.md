## Successes

- **Content architecture designed upfront in Story 3.1 — zero redesigns across 5 stories.** The structure.json + locale missions.json separation was a key design win enabling all subsequent stories.
- **contentLoader caching pattern reused seamlessly in every story** — synchronous loading at startup, zero file I/O at runtime, `getContent()` used in Stories 3.2-3.5 without modification.
- **253 new tests with zero regressions (+127% growth)** — from 199 to 452 tests. Shared test fixtures (`__fixtures__/curriculum.ts`) eliminated repetitive mock setup.
- **Dev notes compounding effect** — by Story 3.5, dev notes included pseudocode, mock patterns, route ordering, and edge cases. Each story was faster than the last.
- **100% retro follow-through (3rd consecutive commitment)** — All 3 action items and all 4 team agreements from Epic 2 retro honored.
- **Review findings acceleration** — dropped from 8 CRITICAL/HIGH in Story 3.1 to polish-only in Stories 3.4-3.5. Team internalized patterns across the epic.
- **Holistic schema planning proven again** — Story 3.1 designed content architecture, Story 3.2 designed DB schema. Both held up perfectly.

## Challenges

- **Foundation story complexity** — Story 3.1 had 19 new files and 8 CRITICAL/HIGH review findings. Foundation stories (2.1 in Epic 2, 3.1 in Epic 3) consistently absorb the most complexity and produce the most review findings. This is structural, not personal.
- **Locale fallback pattern reimplemented 4 times** — Same try-locale/fallback-en/throw-500 pattern copied in Stories 3.2, 3.3, 3.4, 3.5 instead of extracted into shared utility. Story 3.4's review extracted a helper but only for tooltipService.
- **Pre-existing migration issue** — `remove_instagram_auth_provider` migration from Epic 2 needed fixing in Story 3.2 (DROP DEFAULT before enum type change). Dev database reset required.
- **No e2e tests** — All 452 tests are unit/integration with mocks. No tests hit the real database + real content loader together. Mock drift risk grows with each epic.

## Key Insights

1. **Holistic upfront architecture wins again** — Story 3.1's content design held perfectly across 5 stories with zero redesigns. Third consecutive epic validating this pattern.
2. **Pattern acceleration is real and measurable** — Review findings dropped from 8 CRITICAL/HIGH to polish-only. Dev notes compound exponentially, making each successive story faster.
3. **Extract patterns at second occurrence, not later** — Locale fallback was copy-pasted 4 times. New team agreement: extract reusable patterns when they appear a second time.
4. **E2E testing is overdue** — 452 mocked tests aren't sufficient. Real end-to-end testing infrastructure needed before Epic 4 to catch mock drift and performance issues.
