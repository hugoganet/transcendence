## Successes

- **100% backend scope delivery** — All 7 backend stories implemented, reviewed, and passing
- **Test growth: 81 → 199 with zero regressions** — 118 new tests, +146% growth, not a single regression throughout
- **Holistic schema planning in Story 2.1** — Added fields for Stories 2.5 (twoFactorSecret, twoFactorEnabled) and 2.7 (disclaimerAcceptedAt) upfront, eliminating extra migrations
- **Security-first code reviews caught 8 HIGH issues** — XSS in email templates, missing CSRF state, plaintext OAuth tokens, brute-force exposure, email enumeration leaks — all fixed before merge
- **Pattern reuse as force multiplier** — Dev notes from earlier stories explicitly referenced in later stories; by Story 2.7, each story referenced 3-6 prior stories
- **Scope change handled professionally** — Instagram OAuth removal documented, scope adjusted, no disruption
- **10/10 follow-through on Epic 1 retro commitments** — All 5 action items and all 5 preparation tasks completed
- **Velocity increased over epic** — Story 2.1 had most friction (2 debug issues, 3 HIGHs); Story 2.7 had zero debug issues and zero HIGHs
- **Pattern library established** — Express 5 async handling, supertest.agent sessions, vi.hoisted() mocking, AES-256-GCM encryption, rate limiting, email enumeration prevention

## Challenges

- **Test mock complexity scaling** — Affected 4/7 stories. As authService.ts grew, each test file needed more mocks (session.ts, emailService.ts, totpCrypto.ts). The vi.hoisted() pattern helped but didn't eliminate the pain.
- **authService.ts growing too large** — Single file handles register, login, OAuth, password reset, 2FA, session invalidation. Multiple concerns in one module makes testing harder.
- **First-pass security awareness** — Code review caught HIGH security issues in 4/7 stories. Improving trend (3 HIGHs in 2.1, 0 HIGHs by 2.7) but still relying on review as safety net.
- **ESM limitations in test infrastructure** — Early stories (2.1, 2.2) had tests that duplicated handler logic instead of testing real routes due to ESM module loading. Fixed in 2.2 review.
- **Express 5 + Passport.js integration friction** — Story 2.1 absorbed significant pain: body parser ordering, async req.logout(), session regeneration. Later stories benefited but 2.1 was rough.

## Key Insights

1. **Holistic upfront planning is a proven force multiplier** — Schema planning in 2.1 eliminated migrations in 2.5 and 2.7; research tasks from Epic 1 retro enabled better library choices (otpauth over abandoned speakeasy)
2. **Security awareness improves with practice** — HIGH findings went from 3 in Story 2.1 to 0 by Story 2.7. The team internalized encryption, rate limiting, and enumeration prevention patterns.
3. **10/10 retro follow-through validates the retrospective process** — Every commitment from Epic 1 was honored, and it measurably improved Epic 2 execution
4. **Dev notes compound exponentially** — By Story 2.7, each new story referenced 3-6 previous stories' dev notes, eliminating rediscovery of solved problems
